# API Service Integration Demo

Demo repository that collects examples of serverless ingestion and analytics dashboard code for integrating external API services.

The project is intentionally anonymized. Provider names, customer names, cloud account identifiers, database hosts, domains, credentials, and operational documents were replaced with placeholders or removed.

## What This Shows

- AWS Lambda integrations for:
  - POS/sales API ingestion with bearer-token authentication.
  - Invoice API download and XML parsing.
  - Product cost analytics backed by a relational database.
  - Scheduled start/stop automation for cloud instances.
- Streamlit analytics dashboard patterns.
- AWS Secrets Manager usage through placeholder secret names and environment variables.

## Repository Structure

```text
source/
  analytics-dashboard/          Streamlit dashboard examples
  cloud-instance-start-lambda/  Lambda example for starting cloud compute
  cloud-instance-stop-lambda/   Lambda example for stopping cloud compute
  invoice-api-lambda/           Invoice API integration example
  pos-api-lambda/               POS/sales API integration example
  pos-api-secondary-lambda/     Secondary POS/sales API integration example
  product-costing-lambda/       Product cost processing examples
```

## Security Notes

This repository is for demonstration only.

- No real credentials are included.
- Secret names are placeholders such as `demo/database/credentials`.
- Hosts and domains use placeholder values such as `demo-db.example.internal` and `example.com`.
- Removed files include private PDFs, local SSH scripts, CSV exports, generated bytecode, brand images, and raw secret files.

## Local Development

Each subproject keeps its own dependencies and entry points. Start from the README inside `source/analytics-dashboard` for the dashboard, or inspect the individual Lambda folders for focused API integration examples.

Use environment variables or AWS Secrets Manager for any real configuration if adapting these examples.
